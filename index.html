Smart Dailies ‚Äì Automating and Simplifying Reviews

24-Hour View Auto-Filter

Overview

Smart Dailies shows everything submitted in the last 24 hours across every department in one place. Instead of digging through shots or searching manually, you get a clean view of what came in overnight to start your day organized and ready to review.

Complete Workflow

  • Step 1: Opening ShotGrid and finding Smart Dailies
  • Step 2: Adjusting and using filters (24h, 48h, 72h)
  • Step 3: Selecting versions for review by department or milestone
  • Step 4: Creating playlists for Dailies, Postings, or Deliveries
  • Step 5: Running the review session and taking notes
  • Step 6: Submitting notes and tagging artists automatically
  • Step 7: Wrapping up and moving to the Notes App

Key Points

  • Playlist naming: YYYY_MM_DD_DAILIES (e.g., 2025_10_31_DAILIES)
  • Use Task column to confirm department ownership
  • Hold Cmd/Ctrl to select multiple versions
  • Set filter to 48 hours on Mondays to catch weekend work

Notes App ‚Äì Capturing Feedback During Dailies

Live Review Notes Auto-Delivery

Overview

Learn how to capture feedback during dailies using the Notes App. Type notes live during playlist review, and ShotGrid automatically delivers them to artists immediately‚Äîno waiting on recap emails. Notes are logged on the shot and ready for action right away.

What You'll Learn

  • Video 1: Introduction ‚Äî Why we use the Notes App to deliver feedback immediately after review
  • Video 2: Navigating to the Notes Panel inside the playlist viewer
  • Video 3: How to write clear, actionable notes with proper formatting
  • Video 4: Adding CC to ensure the right artists get notes for their department
  • Video 5: Review flow ‚Äî Submit and move to the next shot without stopping

Note Formatting Best Practices

  • Format: note text (artist DEPT)
  • Example: hair cards need more breakup (nico GROOM)
  • Example: too much squash and stretch in character (rae ANIM)
  • Example: reduce rim light intensity (carl LIGHT)
  • One line = one actionable note
  • Always include artist name and department in parentheses
  • Click Submit when done, then move to the next shot

Optional: Flame Suite Review Method

If review is done in Flame instead of ShotGrid: Create a playlist of the latest versions, open the ShotGrid Notes App, enter notes while reviewing in Flame using the same formatting style, CC the correct artists, and submit at the end of the session. This keeps notes consistent regardless of where review happens.

Tracking Work in CG Shots

CG Tracking Status Updates

Overview

Learn how versions are automatically published to ShotGrid with artist and task already assigned. Use the CG Shots page to track work, update statuses after review, and batch mark reviewed versions as Viewed. Production keeps the CG Shots page accurate so the show is always up to date‚Äîno spreadsheets, no GChat messages, no asking artists for updates.

What You'll Learn

  • Video 1: Introduction ‚Äî How ShotGrid replaces spreadsheets and GChat for tracking CG work
  • Video 2: Navigating the CG Shots page and understanding task statuses
  • Video 3: Marking individual versions as Viewed during dailies
  • Video 4: Updating task statuses after dailies (Kicked Back vs Complete)
  • Video 5: Batch marking all reviewed versions as Viewed to keep Smart Dailies clean

Key Concepts

  • Pending Review: Shot needs to be reviewed in dailies
  • Kicked Back: Shot has notes and needs revisions
  • Complete: Shot is approved and finished
  • Viewed: Version has been reviewed and notes have been taken
  • Production adds shots to CG Shots page and updates statuses after each dailies session

Posting Protocol ‚Äì Version Tracking & Playlist Standards

Version Tracking Client Notes Coming Soon

Overview

Learn the discipline behind version tracking on posting days. When work goes to a client with no burnins, the Posting playlist is your only record. This video covers the workflow, naming conventions, and how to keep client notes organized so nobody's guessing when feedback comes back.

What You'll Learn

  • Why version tracking matters ‚Äî the snowball effect
  • Posting playlist naming convention (YYYY_MM_DD_POSTING)
  • The posting workflow ‚Äî create, track, confirm, record
  • Two methods for building your playlist
  • Switching note type from Internal to Client
  • Handling client notes ‚Äî type them or screengrab visual markups
  • Dailies playlist naming convention (YYYY_MM_DD_DAILIES)

Posting Workflow ‚Äî In Order of Importance

  • 1. Create the playlist first thing ‚Äî don't scramble after the fact
  • 2. Confirm with leads before it goes out ‚Äî make sure it matches the edit
  • 3. Trust the playlist when notes come back ‚Äî it's the only record
  • 4. Keep notes organized ‚Äî type them or screengrab, get them on the version
  • Naming: YYYY_MM_DD_POSTING / YYYY_MM_DD_DAILIES
  • Multiple sessions: YYYY_MM_DD_DAILIES_ANIM, YYYY_MM_DD_DAILIES_COMP

Calendar and Milestones

Production Schedule Google Calendar Coming Soon

Overview

Track posting dates, deliveries, milestones, and OOO days using a Google Calendar embedded directly in ShotGrid. The calendar lives on a project page so the whole team sees the same schedule ‚Äî no spreadsheets, no guessing.

How It Works

  • Create a Google Calendar for the project
  • Add posting dates, deliveries, milestones, and OOO
  • Get the shareable embed URL from Google Calendar
  • In ShotGrid, create a Calendar page for the project
  • Add a Web Page Widget and paste the Google Calendar embed URL
  • The calendar now shows live inside ShotGrid

What Goes on the Calendar

  • Postings: Posting EOD 2, Posting EOD 3
  • Deliveries: Final delivery dates
  • Approvals: Post for Approval milestones
  • OOO: Team member out-of-office days
  • Keep it updated ‚Äî this is how the team knows what's coming
